# Editing and Proofreading

Improve and polish written content.

## Purpose

Provide editing assistance for:
- Grammar and spelling
- Clarity and conciseness
- Style consistency
- Flow and structure
- Technical accuracy

## Required Input

- **Text**: Content to edit
- **Type**: Academic, technical, casual
- **Focus**: Grammar, clarity, style, all
- **Constraints**: Word limit, style guide

## Editing Levels

### 1. Proofreading (Light)
- Spelling errors
- Grammar mistakes
- Punctuation
- Typos

### 2. Copyediting (Medium)
- All proofreading items
- Sentence structure
- Word choice
- Consistency
- Formatting

### 3. Substantive Editing (Heavy)
- All copyediting items
- Paragraph organization
- Argument flow
- Content suggestions
- Structural changes

## Process

1. **Read through once**
   - Understand the message
   - Note overall issues
   - Identify the voice

2. **Edit systematically**
   - Work section by section
   - Address one type of issue at a time
   - Preserve author's voice

3. **Review changes**
   - Ensure meaning preserved
   - Check consistency
   - Verify improvements

## Common Improvements

### Conciseness
```
Before: "It is important to note that..."
After: "Notably,..." or just state the point

Before: "In order to"
After: "To"

Before: "Due to the fact that"
After: "Because"
```

### Clarity
```
Before: "This affects the results."
After: "Participant dropout affects the power estimates."

Before: "It was found that..."
After: "We found..." or "[Subject] showed..."
```

### Academic Style
```
Before: "We got significant results."
After: "Results reached statistical significance."

Before: "The data shows..."
After: "The data show..." (data is plural)
```

## Style Guides

### Academic (APA-ish)
- Active voice preferred
- Past tense for results
- Formal but readable
- Avoid "very," "really"

### Technical
- Precise terminology
- Consistent naming
- Clear definitions
- Step-by-step logic

### General Professional
- Clear and direct
- Appropriate formality
- Reader-focused
- Action-oriented

## Examples

**Request**: "Proofread this abstract for spelling and grammar"

**Request**: "Edit for conciseness - need to cut 200 words"

**Request**: "Check this methods section for clarity"

**Request**: "Polish this email before sending to collaborators"

## Tips

- Read aloud to catch awkward phrasing
- Cut unnecessary words ruthlessly
- Keep sentences under 25 words
- One idea per paragraph
- Use transitions between sections
- Verify technical terms are correct

## Track Changes Note

When reviewing edits:
- Accept obvious corrections
- Consider style suggestions
- Keep your authentic voice
- Ask about unclear changes
